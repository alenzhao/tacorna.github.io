TACO: Large scale transcriptome meta-assembly for RNA-Seq

Transcriptome assemblers reconstruct full-length transcripts from the short sequence fragments generated by RNA-Seq. Large consortia such as TCGA, GTex, ENCODE, the Cancer Cell Line Encyclopedia (CCLE), and many others have performed RNA-Seq on thousands of human tissues and cell lines, providing an unparalleled resource for investigating transcriptional diversity and complexity. Towards this end, we present Transcriptome Assemblies Combined into One (TACO), an algorithm that reconstructs a consensus transcriptome from a large collection of individual assemblies. TACO employs change point detection to break apart complex loci and correctly delineate transcript start and end sites, and a dynamic programming approach to assemble transcripts from a network of splicing patterns. TACO vastly outperforms meta-assembly software tools such as Cuffmerge and Stringtie merge. See the results of our comparison analysis here.

Methods

Transcriptome meta-assembly protocol

  1. Sequence reads from each dataset are aligned to the reference genome using STAR, HISAT2, or another RNA-Seq aligner
  2. Read alignments (SAM/BAM/CRAM format) from each dataset are assembled into transfrags using Cufflinks, StringTie, or another ab initio transcriptome assembly tool.
  3. Meta-assembly is performed on the collection of ab initio assemblies (GTF format) by TACO, resulting in a consensus merged transcriptome
  4. Downstream analysis of gene expression, conservation, variation, and functionality can be performed

Results

Varying the number of input assemblies

Performance metrics for TACO, Cuffmerge, and StringTie when merging different numbers of input assemblies. Recall (i.e., sensitivity), precision and the F-measure for all three tools were assessed for splicing patterns, splice junctions, and bases. Twenty batches were randomly selected at each number of assemblies. Points represent the mean statistic across the 20 runs, error bars represent the 95% confidence interval.


Varying isoform fraction cutoff

The parameter --isoform-frac FRAC configures TACO to only report isoforms with abundance ratio more than FRAC of the most abundant transcript in the locus

The precision-recall (PR) plots depict the performance for the three merging tools at 50 different isoform fraction cutoffs ranging from 0.001-0.999. Statistics describe the merger of the 55 breast cancer cell lines in the CCLE.



Performance for highly expressed transcripts

The precision-recall (PR) plots depict the performance for the three merging tools across subsets of the top 500, 1k, 1.5k, 2k, 3k, 5k, 7k, 10k, 12k, 15k, 18k, 20k, 22k, 25k, 27k, and 30k highest expressed transcripts in the merger of 55 breast cancer cell lines from the CCLE



Fraction of read-through transcripts

Scatterplot depicting the fraction of read-through genes produced in the meta-assembly from the three merge tools. Gene designated as being a read-through if it contains at least one transcript isoform bearing exonic overlap with two separate protein-coding genes on the same strand. Fraction of read-throughs was determined for assemblies at 50 different isoform fraction cutoffs ranging from 0.001-0.999.

Installation

Download Software

Download the latest TACO release here

Install dependencies

TACO requires several dependencies:

  • Python 2.7
  • Numpy
  • Cython (required to compile C extension modules)

We developed and tested TACO on Mac OSX and Linux using the Enthought Canopy Python Distribution. If you do not already have a Python distribution installed on your system, we recommend this.

Build

Unpack the release file (.tar.gz or .zip) into a local direectory SOURCE_DIR. For example: 

$ tar -zxvf taco-X.Y.Z.tar.gz

Build and install the software into directory INSTALL_DIR 

$ cd SOURCE_DIR
$ python setup.py install --prefix INSTALL_DIR

Setup your Python environment

The TACO program will be installed to INSTALL_DIR/bin/taco_run.py. Associate modules will be available at INSTALL_DIR/lib/python2.7/site-packages

If you installed TACO into a non-standard directory, your PYTHONPATH environment variable must include the TACO package directory. Using bash shell:

$ export PYTHONPATH=INSTALL_DIR/lib/python2.7/site-packages:$PYTHONPATH

Running TACO

Run TACO from the command line as follows:

$ INSTALL_DIR/bin/taco_run.py <options> <gtf_files.txt>

A detailed description of the parameters used to control TACO is below:

Arguments

<gtf_files.txt>

This required input to TACO is a text file containing paths to files in the GTF 2.2 format, one per line, produced by an ab initio assembler. For example, the text file passed to TACO could look like this:

/path/to/my/assemblies/mcf7.gtf
/path/to/my/assemblies/bt474.gtf
/path/to/my/assemblies/hela.gtf

General Options

-h/--help

Prints the help message and exits

-v/--verbose

Enables detailed logging for debugging purposes

-p/--num-processes N

Run TACO in parallel mode with N processes [default=1]

-o/--output-dir DIR

Directory where output files will be stored (must not already exist)

--gtf-expr-attr ATTR

GTF attribute field containing expression estimate [default=FPKM]

--min-transfrag-length N

Filter input transfrags with length < N prior to assembly [default=200]

--min-expr X

Filter input transfrags with expression < X (gtf-expr-attr parameter) prior to assembly [default=0.5]

--isoform-frac FRAC

Report transcript isoforms with expression fraction >=FRAC relative to the highest expressed gene [default=0.05]

Advanced Options

TACO has additional advanced options that can be used to configure its behavior. We recommend that users leave these options at their default settings.
